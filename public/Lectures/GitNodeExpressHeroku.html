HTML5 APIs: Git, Node, Express, and Heroku

Git

Git is a version control (a.k.a. source control management) system. All serious software projects use some VCS, and Git is one of the most modern and popular. Version control allows developers to keep track of the changes and versions of their code as they work. It facilitates collaboration. It also serves as a back-up system.

The official Git Web site, linked above, has extensive documentation. Because it is widely used, you can find tutorials and answers to most questions by searching the Web. The book by Scott Chacon & Ben Straub, Pro Git, Apress, 2014 (9781484200773 also free online), is very good.

Node.js

Node.js is a command-line interpreter for JavaScript, based on the excellent V8 engine found in Google's Chrome and Chromium browsers. It allows desktops and servers to run JavaScript outside of the browser environment.

Node.js comes with many useful standard libraries, a.k.a. modules, which make it a good platform for many purposes. It is most commonly used to run Web servers, with the help of certain modules.

There is a wealth of books and resources about Node.js.

Express

Express is a Web server framework that runs on Node.js. It has a very modular design, meaning that there is a large collection of modules that are typically added on to Express to enable the features one needs.

Many of the books and articles about Node.js are also about Express, whose official Web site includes a list of books and blogs that is a reasonable starting point.

Heroku

Heroku is a platform-as-a-service (PaaS) provider, which means that it provides a platform where developers can deploy Web applications. Heroku supports applications written in a variety of languages and frameworks, including Ruby on Rails, Node.js, Java, Python, and PHP.

Pricing varies depending on the scale of the application's resource requirements. For small projects where performance is not an issue, the service is free.
